# 战斗系统设计文档

## 概览

《末世行商》战斗系统包含两个独立模式，分别对应不同的游戏场景：

| 模式 | 触发场景 | 玩法原型 | 核心体验 |
|------|---------|---------|---------|
| **车载迎击（Ambush）** | 行驶途中遭遇追击 | FTL 式战术决策 | 高压逃脱，资源博弈 |
| **资源点探索（Explore）** | 途经废墟资源点 | Roguelike 房间探索 | 风险收益，搜刮取舍 |

两种模式**不可同时发生**，均在行驶阶段触发，战斗期间货车停车。

---

## 一、触发流程

```
行驶中 → 随机事件触发
  ↓
event_executor.lua 执行操作：
  "start_combat:ambush_light"  → state.flow.pending_combat = "ambush_light"
  "start_explore:old_clinic"   → state.flow.pending_explore = "old_clinic"
  ↓
main.lua 主循环检测（仅在首页 home 时）：
  pending_combat → Router.navigate("ambush", { enemy_id = ... })
  pending_explore → Router.navigate("explore", { room_id = ... })
  ↓
对应 UI 页面创建战斗实例，战斗开始
  ↓
战斗结束 → combat_result.lua 回写状态 → 返回首页继续行驶
```

**关键规则**：
- 战斗/探索页面期间 `truck_stopped = true`，货车不推进行驶
- pending 标记在进入战斗页面时被清除
- 事件结果面板中的战斗回顾支持滚动查看

## 二、车载迎击（Ambush）

### 2.1 核心机制

玩家扮演驾驶员林砾，每回合选择一个战术，陶夏自动开火。目标是在有限回合内**逃脱或击退追兵**。

**状态变量**：
- **威胁值（threat）**：初始由敌人模板决定，每回合被陶夏火力削减。影响敌人攻击伤害的倍率
- **逃脱进度（escape_progress）**：每回合根据战术增长，达到阈值即脱离
- **回合上限（rounds_total）**：战斗持续的最大回合数

### 2.2 敌人模板

| 模板 ID | 名称 | 初始威胁 | 攻击范围 | 回合 | 逃脱阈值 | 击退战利品 |
|---------|------|---------|---------|------|---------|-----------|
| `ambush_light` | 散兵劫匪 | 40 | 3-8 | 3 | 70 | 金属碎片×1 (30%) |
| `ambush_medium` | 武装车队 | 65 | 6-14 | 4 | 120 | 弹药×1 (25%), 金属碎片×2 (40%) |
| `ambush_heavy` | 赏金猎人 | 85 | 10-20 | 4 | 120 | 弹药×2 (30%), 电路板×1 (20%) |

### 2.3 战术系统

每回合玩家从 4 种战术中选择一种：

| 战术 | 图标 | 逃脱进度 | 受伤倍率 | 油耗 | 特殊效果 |
|------|------|---------|---------|------|---------|
| 加速突破 | 🚀 | +35 | ×1.2 | 3 | 高逃脱，高风险 |
| 稳车射击 | 🎯 | +20 | ×0.8 | 0 | 有弹药时额外削减威胁+15 |
| 急转规避 | ↩️ | +15 | ×0.3 | 2 | 大幅减伤 |
| 烟幕撤离 | 💨 | 直接逃脱 | ×0.0 | 0 | 消耗1个烟雾弹，立刻脱离 |

### 2.4 弹药与火力

**弹药消耗规则**：每回合消耗 1 发弹药（后期弹药充裕，每回合消耗更贴合节奏）。

| 条件 | 火力（每回合削减威胁） |
|------|---------------------|
| 有弹药 | 12（稳车射击时额外+15 = 27） |
| 无弹药 | 3 |
| 陶夏受伤 | 火力×0.5 |
| 车顶机枪加成 | 火力×模块倍率（Lv2=1.3, Lv3=1.5） |

### 2.5 伤害计算

```
base_dmg = random(atk_min, atk_max)
threat_factor = current_threat / 100
dmg = floor(base_dmg × threat_factor × tactic.dmg_mult + 0.5)
```

伤害直接扣减货车耐久（`state.truck.durability`）。

### 2.6 战斗结算

三种结局：

| 结局 | 条件 | 后果 |
|------|------|------|
| **escaped（脱离）** | 逃脱进度 ≥ 阈值 | 正常结束 |
| **repelled（击退）** | 回合耗尽且威胁 ≤ 20 | 获得战利品 |
| **wrecked（重创）** | 货车耐久归零 | 掉货+林砾疲劳 |

**掉货机制**（重创时触发）：
- 遍历所有货物，价值加权计算掉落概率
- 公式：`chance = clamp(0.6 - base_price × 0.008, 0.15, 0.55)`
- 廉价货物（水 10元）掉率 52%，贵重货物（燃料电池 50元）掉率 20%
- 每种最多掉 2 个

**战后效果**：
- 重创：林砾获得"疲劳"状态
- 累计伤害 ≥ 15：40% 概率随机一人受伤

### 2.7 雷达模块影响

| 雷达等级 | 效果 |
|---------|------|
| Lv1 | 无直接战斗效果（探索模式有加成） |
| Lv3 | **敌情预警**：初始威胁值 ×0.8（-20%） |

### 2.8 车顶机枪模块影响

| 机枪等级 | 火力倍率 | 逃脱阈值折扣（repel_bonus） |
|---------|---------|---------------------------|
| Lv1 | ×1.0 | 0% |
| Lv2 | ×1.3 | 30% |
| Lv3 | ×1.5 | 50% |

逃脱阈值折扣公式：`threshold = floor(原始阈值 × (1 - repel_bonus) + 0.5)`

## 三、资源点探索（Explore）

### 3.1 核心机制

玩家操控林砾和陶夏进入废墟资源点，逐个搜刮容器。搜刮后有概率遭遇敌人，可选择战斗或逃跑。随时可撤离。

**核心循环**：
```
进入 → 选择搜刮一个容器 → 获得物品
  ↓                          ↓
  ←———— 安全 ←————————————— 判定遭遇
  ↓                          ↓ 遭遇敌人
  ↓                     战斗 or 逃跑
  ↓                          ↓
  ←—————————————————————— 继续探索
  ↓
  撤离（全部搜完或主动撤离）
```

### 3.2 房间模板

| 房间 ID | 名称 | 搜刮点 | 敌人 | 危险概率 | 危险伤害 | 事件触发 |
|---------|------|-------|------|---------|---------|---------|
| `abandoned_warehouse` | 废弃仓库 | 铁箱、木箱 | 变异鼠群(20HP/4ATK) | 30% | 5 | EVT_038 |
| `old_clinic` | 旧诊所 | 药柜、急救箱 | 流浪者(15HP/3ATK) | 20% | 3 | EVT_057 |
| `military_outpost` | 军事哨站 | 弹药箱、储物柜 | 残兵(30HP/8ATK), 巡逻机器人(25HP/6ATK) | 45% | 8 | EVT_058 |
| `radar_station` | 废弃雷达站 | 信号柜、工具箱 | 野狗群(18HP/5ATK) | 35% | 6 | EVT_059 (需雷达) |

### 3.3 玩家体力

体力（HP）动态计算，取决于货车状态和雷达模块：

```
base_hp = 40 + floor(truck.durability × 0.1)
radar_bonus = 0 or 0.15（雷达 Lv1+）
hp_max = floor(base_hp × (1 + radar_bonus) + 0.5)
```

典型值：耐久 100 时 base_hp=50，有雷达 Lv1 时 hp_max=57。

### 3.4 搜刮机制

- 每个容器有独立的物品表（id + count + chance）
- **延迟入包**：搜到的物品先存入 `items_found`，不立刻写入游戏状态
- 撤离时按结局比例提交到 `state.truck.cargo`
- 雷达 Lv1+ 提升搜刮发现概率 +15%（上限 95%）

### 3.5 探索战斗

简化回合制，每回合：

1. **玩家攻击**：林砾近战（12ATK，受伤时×0.6）+ 陶夏辅助（有弹药 8ATK / 无弹药 2ATK，受伤时×0.5）
2. 攻击含随机浮动：`total_atk × (0.8 + random × 0.4)`
3. **敌人反击**：`atk × (0.7 + random × 0.6)`
4. **弹药消耗**：每回合战斗消耗 1 发弹药（有弹药时）

### 3.6 医疗包使用

探索期间可使用医疗包（`medicine`）：
- 消耗 1 个医疗包，恢复 20 HP
- 仅在 HP < HP_MAX 且有库存时可用
- 在探索阶段（非战斗中）使用

### 3.7 结局与搜刮物保留

| 结局 | 条件 | 搜刮物保留比例 | 额外后果 |
|------|------|-------------|---------|
| **success（成功）** | 主动撤离 | 100% | 受伤×0.3 映射到耐久 |
| **fled（逃跑）** | 遭遇时选择逃跑 | 50% | 被追击一次（atk×0.5 耐久伤害） |
| **wounded_retreat（负伤撤退）** | HP 归零 | 30% | 伤害×0.5 映射到耐久，林砾受伤 |

**搜刮物保留公式**：`actual = floor(found_count × fraction + 0.5)`

### 3.8 战后效果

- 军事哨站特有：若发生过遭遇，20% 概率林砾中毒
- 负伤撤退：林砾获得"受伤"状态

## 五、角色状态对战斗的影响

| 状态 | 迎击影响 | 探索影响 |
|------|---------|---------|
| 林砾受伤（wounded） | 驾驶能力-25%（逃脱进度×0.75） | 攻击力×0.6 |
| 陶夏受伤（wounded） | 火力×0.5 | 辅助攻击×0.5 |
| 林砾疲劳（fatigued） | 无直接影响（影响其他系统） | 无直接影响 |

## 六、物品在战斗中的作用

| 物品 | 迎击用途 | 探索用途 |
|------|---------|---------|
| **弹药（ammo）** | 每回合消耗1发，提供12点火力 | 每回合战斗消耗1发，陶夏辅助攻击8→2 |
| **烟雾弹（smoke_bomb）** | 消耗1个，立即脱离战斗 | 不可用 |
| **医疗包（medicine）** | 不可用 | 探索中使用，恢复20HP |

## 七、事件配置

战斗/探索通过随机事件系统触发，配置在 `assets/configs/guaji_random_events.json` 中。

事件操作格式：
```json
{
  "ops": ["start_combat:ambush_light"]    // 触发车载迎击
}
{
  "ops": ["start_explore:old_clinic"]     // 触发探索
}
```

已配置的战斗触发事件：
- **EVT_038**：危险路段遭遇袭击 → 迎击（ambush_light/medium/heavy 随段位）
- **EVT_057**：废弃诊所 → 探索（old_clinic）
- **EVT_058**：军事哨站废墟 → 探索（military_outpost）
- **EVT_059**：锈迹斑斑的信号塔 → 探索（radar_station，需 has_radar 标志）

## 八、代码结构

```
scripts/combat/
├── combat_config.lua     # 敌人模板、战术定义、房间模板、数值参数
├── ambush.lua            # 车载迎击逻辑（create/execute_round/get_summary）

├── screen_ambush.lua     # 迎击 UI（威胁条+逃脱进度+战术按钮+战斗回顾）
└── screen_explore.lua    # 探索 UI（房间描述+搜刮点+战斗交互+撤离）
```

## 九、数值平衡备注

### 弹药经济
- 弹药单价 45 信用点，每场迎击消耗 1-4 发（取决于回合数）
- 后期弹药充裕，每回合消耗不构成压力，但零弹药惩罚显著（火力 12→3）
- 探索战斗同样每回合消耗 1 发

### 逃脱难度梯度
- 散兵（阈值70）：纯加速 2 回合即可脱离（35×2=70）
- 武装车队（阈值120）：需 4 回合加速或组合战术
- 赏金猎人（阈值120）：同上但威胁更高，受伤更多

### 探索风险收益
- 旧诊所：低风险低收益（医疗包为主）
- 军事哨站：高风险高收益（弹药+电路板，但敌人强力）
- 成功撤离保留 100%，逃跑 50%，负伤 30% —— 鼓励见好就收
