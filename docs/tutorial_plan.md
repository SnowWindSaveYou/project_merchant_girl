# 开场引导计划

> 状态：待实施 | 优先级：游戏稳定后

## 触发时机

残骸序幕（SD_PROLOGUE_01）结束 → **引导流程** → 首次篝火自然触发第一晚（SD_PROLOGUE_02）

```
新游戏 → 残骸(自动) → home(引导开始) → … → 首次篝火 → 第一晚(自然)
```

## 当前状态

- SD_PROLOGUE_01 已改为自动触发（main.lua 检测 flag）
- 01→02 的强制链接已移除，等引导系统做好后再衔接
- SD_PROLOGUE_02 会在玩家首次使用篝火时自然出现（weight:999 最高优先）

## 引导应覆盖的核心循环

### Step 1: 认识首页
- 高亮关键按钮（商店、地图、货仓）
- 简短文字提示："看看商店里有什么可以带上路的。"

### Step 2: 商店购买
- 引导进入商店，购买首批货物
- 玩家初始 200 credits，最便宜的 water 10cr/个
- 建议引导买 food_can（15cr），因为后续篝火需要消耗

### Step 3: 查看地图 & 选择目的地
- 引导打开地图，高亮可前往的节点（crossroads / tower）
- 解释路线规划基础

### Step 4: 出发
- 引导确认出发，解释行程中的等待/事件

### Step 5: 到达 & 结算
- 首趟结束后展示结算画面
- 引导理解收益

### Step 6: 篝火（可选）
- 如果货仓有 food_can，提示"可以在篝火旁休息"
- 触发 SD_PROLOGUE_02（自然流程，非强制）

## 技术方案备选

### A. 对话式引导（简单）
- 用现有 Gal 对话系统，陶夏/林砾的对话形式指引
- 每个步骤前弹出一段简短对话
- 优点：零 UI 开发，复用现有系统
- 缺点：无法高亮具体按钮

### B. 覆盖层引导（标准）
- 半透明遮罩 + 高亮目标区域 + 提示文字
- 需要 NanoVG 或 UI 库实现覆盖层组件
- 优点：标准教程体验
- 缺点：需要新组件开发

### C. 混合方案（推荐）
- 对话提示（陶夏说"去商店看看"）+ 按钮闪烁/颜色提示
- 按钮上加标记（如 "← 点这里"）
- 折中复杂度，效果够用

## 依赖

- 游戏核心循环稳定（商店/地图/行程/结算无重大 bug）
- UI 组件支持条件性高亮或标记
- Flag 系统追踪引导进度（tutorial_step_1_done 等）

## 篝火资源门槛问题

SD_PROLOGUE_02 需要篝火触发，篝火需要 1 food_can 或 1 fuel_cell。
引导中应确保玩家购买 food_can，或考虑：
- 方案 A：引导中强制建议购买 food_can
- 方案 B：首次篝火免资源消耗（加 tutorial flag 判断）
- 方案 C：新游戏初始货仓自带 1 food_can
