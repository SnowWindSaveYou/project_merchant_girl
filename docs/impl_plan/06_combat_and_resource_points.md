# Phase 06 — 车载迎击与资源点探索战

## 本阶段目标

把两种战斗接进主循环，但仍然保持“战斗是服务跑商，不是喧宾夺主”。

这阶段完成后，玩家应该能明确感受到：

- 路上遇袭时要快速决策
- 停车探索时要搜刮并撤离

---

## 本阶段只关心什么

- 车载迎击
- 资源点探索战
- 战备消耗品
- 战斗结果回写主循环

## 本阶段不要做什么

- 不做复杂武器养成
- 不做大地图实时战斗
- 不做长时间刷怪关卡

---

## 模式 A：车载迎击

### 目标体验

玩家在 10~20 秒内做 2~3 次判断：

- 加速
- 稳车
- 规避
- 烟幕撤离

### 实现重点

- 战斗仍发生在行驶页上
- 不切独立场景
- 陶夏火力自动执行
- 玩家只做战术决策

### 最小数据

- 敌人压迫值
- 货车耐久
- 脱离进度
- 当前战术按钮冷却（如果有）

### 结果必须回写

- 耐久变化
- 燃料变化
- 耗时变化
- 是否掉货
- 是否改道

---

## 模式 B：资源点探索战

### 目标体验

玩家不是清图，而是进去拿资源后撤。

### 首版最小场景

只做一个模板房间：

- 2 个货箱
- 1 个敌人或 2 个弱敌
- 1 个撤离点
- 1 个互动机关

### 首版最低要求

- 进入资源点
- 移动或简化位移
- 搜刮
- 撤离
- 战斗结果回写外层状态

---

## 战备资源接法

### 核心规则：战备消耗品 = 货舱商品

战备消耗品**不设独立战备背包**，直接从货舱商品中消耗。弹药和烟雾弹本身也是可交易商品，玩家需要在"卖掉赚钱"和"留着保命"之间权衡。

**已实现基础设施**：
- `scripts/economy/item_use.lua` — 统一的 `use()`/`consume()`/`add()` 接口
- `scripts/events/event_executor.lua` — 支持 `consume_goods:xxx:n` 和 `add_goods:xxx:n` 字符串操作
- `scripts/ui/screen_cargo.lua` — 货舱界面已支持可使用物品的"使用"按钮

### 消耗品清单与详细规则

| 消耗品 | 商品 ID | 获得方式 | 消耗时机 | 效果 | 每次消耗 | 可叠加 |
|--------|---------|---------|---------|------|---------|--------|
| **弹药链** | `ammo` | 聚落购买、事件奖励 | 车载迎击（自动） | 陶夏机枪持续射击；无弹药时火力大幅下降 | 1/次战斗 | 是（多带多用） |
| **烟雾弹** | `smoke_bomb` | 聚落购买、事件奖励 | 车载迎击（主动） | 立即脱离战斗，保全货物，损失时间 | 1/次使用 | 是 |
| **医疗包** | `medicine` | 聚落购买、事件奖励 | 战斗后/随时 | 清除 1 个角色负面状态 | 1/次使用 | 是 |
| **照明棒** | 待定 | 聚落购买 | 探索战进入时 | 降低暗区风险 | 1/次探索 | 是 |

### 实现接入要点

1. **车载迎击开始时**：检查 `state.truck.cargo` 中 `ammo` 数量 → 决定火力等级
2. **烟幕撤离按钮**：检查 `smoke_bomb > 0`，使用后调 `ItemUse.consume(state, "smoke_bomb", 1)`
3. **医疗包**：已在 `item_use.lua` 注册，待负面状态实装后扩展 effect 函数
4. **战斗结算**：通过 `event_executor` 的 `consume_goods`/`add_goods` 批量处理奖惩

## 角色负面状态系统

### 状态定义

| 状态 | ID | 触发来源 | 效果 | 清除方式 |
|------|-----|---------|------|---------|
| **疲劳** | `fatigued` | 连续行驶过久、战斗后 | 行驶速度 -10%，事件判定略差 | 医疗包 / 据点休整 |
| **受伤** | `wounded` | 战斗失利、危险事件 | 对应角色技能暂时失效 | 医疗包 / 据点休整 |
| **中毒** | `poisoned` | 特定事件、辐射区域 | 每段行程额外消耗耐久 -5 | 医疗包 |
| **士气低落** | `demoralized` | 连续订单超时、货损严重 | 好感获取 -20%，交易价格略差 | 到达高好感据点自动清除 |

### 实现方案

```lua
-- state.lua 中为角色增加状态字段
characters = {
    linli  = { status = {} },  -- 负面状态 ID 数组
    taoxia = { status = {} },
}

-- 施加状态（同一状态不叠加）
function add_status(char, status_id)
    for _, s in ipairs(char.status) do
        if s == status_id then return end
    end
    table.insert(char.status, status_id)
end

-- 清除状态（医疗包清除第一个，据点休整清除全部）
function clear_one_status(char)
    if #char.status > 0 then
        table.remove(char.status, 1)
    end
end

function clear_all_status(char)
    char.status = {}
end
```

### 设计原则

- 负面状态**有感但不致命**，不会导致无法继续游戏
- 同一状态不叠加
- 清除优先级：medicine 清除任意一个 → 据点休整清除全部

## 和主循环的连接点

### 行驶中遇袭

- 从事件系统或路线节点进入
- 暂停普通行驶推进
- 进入战斗子状态
- 战斗完成后返回 `TRAVELLING`

### 资源点探索

- 从大地图 `resource` 节点主动进入
- 暂时脱离普通路线
- 完成后返回外层地图 / 行驶页

并补充限制：

- 不是所有节点都可进入探索战
- `settlement` 节点默认不进入探索战
- `resource` 节点可配置“可探索 / 已清空 / 限时刷新”状态

## 首版一定要控制复杂度

### 车载迎击不要做

- 手动瞄准
- 多敌人 AI 路径系统
- 多武器切换

### 探索战不要做

- 大地图追逐
- 复杂掩体系统
- 大量敌人刷新波次

只做最小可玩，再迭代。
