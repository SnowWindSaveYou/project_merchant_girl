# Phase 08 — 联调、性能、上线前准备

## 本阶段目标

把前面已经做好的系统接起来，清掉最影响上线的问题。

这阶段不再新增大系统，重点是：

- 联调
- 稳定性
- 性能
- 数据校验
- 上线前最小运维准备

---

## 本阶段只关心什么

- 系统联调
- 性能热点
- 存档稳定性
- 云端同步稳定性
- 配置正确性
- 基础埋点

## 本阶段不要做什么

- 不临时加大型新系统
- 不重写基础架构
- 不把测试期问题拖到上线后再说

---

## 联调顺序

按这个顺序接，不要全一起开：

1. 核心循环 + UI
2. 贸易 + 路线
3. 事件 + 旗标
4. 战斗 + 主循环回写
5. 聚落 + 好感 + 关系
6. 存档 + 云同步

每接完一层，就做一轮回归。

---

## 重点测试清单

### 存档相关

- 行驶中退出再进入
- 到站前退出再进入
- 事件弹窗中退出再进入
- 网络断开时云同步失败
- 本地缓存比云端新 / 云端比本地新

### 经济相关

- 订单结算后资源是否正确
- 价格修正是否越界
- 货物数量是否出现负数

### 战斗相关

- 车载迎击退出后是否回到正确状态
- 资源点撤离后是否回到正确页面
- 战斗失败是否导致状态错乱

---

## 性能优先排查点

### NanoVG

- 是否有重复加载图片
- 是否有每帧构造大量临时字符串
- 滚动列表是否绘制了不可见项
- 是否缺少 `Save / Restore` 导致状态污染

### Lua

- 高频 update 中是否频繁创建 table
- 是否有事件筛选全表扫描过多
- 是否有 UI 页面每帧重复计算固定布局

---

## 配置校验

上线前至少做一次完整配置扫描：

- 重复 ID
- 缺失引用
- 非法枚举值
- 路线引用不存在的聚落
- 事件引用不存在的结果组
- 商品引用不存在的分类

这一步最好做成启动时开发模式自动检查。

---

## 埋点最低要求

至少记录：

- 新手阶段流失点
- 首次跑商完成率
- 首次战斗完成率
- 首次升级完成率
- 云同步失败率

没有这些数据，上线后会很难判断问题在哪。

---

## 上线前必须确认

- 本地缓存不会把状态写坏
- 云端摘要同步失败不会卡死主流程
- 离线补偿不会异常爆资源
- 关键页面在主流分辨率下可用
- 首 2 小时流程能完整跑通

---

## 通过标准

- [ ] 主循环、贸易、事件、战斗、成长能完整串起来
- [ ] 主要存档和恢复场景已回归
- [ ] 云同步失败有降级处理
- [ ] 关键配置已做一致性检查
- [ ] 首 2 小时流程能稳定跑通
- [ ] 已有最基本埋点可看
