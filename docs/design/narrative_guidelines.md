# 《末世行商》叙事撰写规范

> **面向 AI 智能体的剧情编写守则**
>
> 本文档规定了编写主线对话、篝火对话、NPC 对话、事件文本时必须遵守的规则。
> 违反这些规则会导致叙事逻辑矛盾、调性失控或玩家体验断裂。

---

## 一、叙事通道职责划分

游戏共有 **六个叙事通道**，各自有明确的触发场景、内容边界和体验定位。编写内容前必须先确认该内容属于哪个通道。

### 1.1 总览

| # | 通道 | 触发方式 | 触发场景 | 体验定位 | 配置来源 |
|---|------|---------|---------|---------|---------|
| 1 | **主线对话** | 自动（到达拦截/章节推进） | 到达聚落、行程里程碑 | 世界推进、外部冲突、重大选择 | `story_dialogues.json` |
| 2 | **篝火对话** | 玩家主动（消耗食物/燃料） | 停车休息时 | 双主角关系、回忆、情感深化 | `campfire_dialogues.json` |
| 3 | **NPC 对话** | 玩家主动（点击"交谈"） | 聚落内 | 具名 NPC 关系、任务线索、聚落历史 | `npc_dialogues.json` |
| 4 | **到处晃晃** | 玩家主动（消耗饮用水） | 聚落内 | 聚落日常生活、地方风味、低风险探索 | `stroll_scenes.json` |
| 5 | **随机事件** | 自动弹出（行驶中打断） | 开车途中 | 突发危险、路边交易、资源搜刮、聚落危机 | `waypoint_events.json` / `settlement_events.json` |
| 6 | **驾驶闲聊 & 收音机** | 自动播放（不打断行驶） | 开车途中 | 旅途氛围、角色碎语、世界观点缀 | `chatter.json` / `radio_broadcasts.json` |

### 1.1.1 跨通道机制：信件系统

> 详细设计见 `docs/design/letter_system.md`

信件系统**不是第七个叙事通道**，而是一个跨通道机制——它为主线、NPC对话、订单系统提供内容输入。

| 信件功能 | 输入到哪个通道 | 说明 |
|---------|--------------|------|
| NPC来信 | 主线对话 | 作为悬念钩子，牵引玩家前往某聚落（"韩策来信说数据有异常"） |
| 来信内容触发后续 | NPC对话 | 下次见面时可以聊信中的话题 |
| 送信委托 | 订单系统 | NPC在交易所挂出送信任务，主角接取并送达 |
| 收信后的讨论 | 篝火对话 | 两人私下讨论来信内容（"回响"，不推进） |
| 收信后的感想 | 驾驶闲聊 | 轻量伴随式评论 |

**信件不做的事**：不打断行驶（雪冬只在节点出现）、不替代面对面互动、不传递超出NPC信息权限的内容。

### 1.1.2 悬念钩子与升温机制

> 详细设计见 `docs/design/suspense_hooks.md`

悬念钩子是贯穿各叙事通道的**连续性设计**，解决章间自由行商期的叙事动力衰减问题。

**三钩原则**：每个章节末尾至少设下 3 个不同层级的新钩子——

| 层级 | 名称 | 机制 |
|------|------|------|
| L1 | 信息缺口 | 给一半藏一半，让玩家想知道答案 |
| L2 | 预期偏差 | 建立预期A暗示B，让玩家想验证猜测 |
| L3 | 情感悬置 | 未完成的角色情感状态，让玩家想看到结果 |

**升温机制**：章间自由行商期（5~15 趟），通过闲聊/广播/信件/篝火定期提醒已有钩子，防止遗忘。升温内容**只提醒不推进**。

**编写新章节/新对话时必须检查**：
- 本章末尾是否满足三钩原则？
- 是否有钩子在设下后超过 10 趟行程未被任何通道提及？
- 升温内容是否放在了正确的通道中？

### 1.2 各通道职责边界

#### 主线对话 (story_dialogues)

| 负责 | 不负责 |
|------|--------|
| 世界推进、外部冲突、NPC 互动 | 两人的情感日常、关系升温 |
| 有后果的选择（设置影响后续的 flag） | 纯氛围、纯关系数值 |
| 引入新势力、新线索、新危机 | 日常吐槽、回忆往事 |

#### 篝火对话 (campfire_dialogues)

| 负责 | 不负责 |
|------|--------|
| 两人的情感进展、回忆、日常互动 | 推进世界线、引入新势力 |
| 关系深化、记忆碎片解锁 | 触发重大事件 |
| "回响"已发生的主线事件（讨论，不推进） | 传递新的世界信息 |

#### NPC 对话 (npc_dialogues)

| 负责 | 不负责 |
|------|--------|
| NPC 个性展现、聚落信息 | 代替主角做决定 |
| 任务线索、好感度门控的深层信息 | 跨聚落传递确切情报（见第三节） |
| 具名角色的个人故事线 | 处理匿名路人互动 |

#### 到处晃晃 (stroll_scenes)

| 负责 | 不负责 |
|------|--------|
| 聚落日常生活切片（浇水、祷告、市集） | 高风险危机事件 |
| 地方文化和环境细节 | 主线推进、任务触发 |
| 低风险的小互动（帮忙修管道、看孩子玩耍） | 战斗、大额资源变动 |

#### 随机事件 (waypoint_events / settlement_events)

| 负责 | 不负责 |
|------|--------|
| **战斗/危险**：野狗、盗匪、车辆故障、辐射区 | 关系培养、记忆碎片 |
| **路边交易**：拾荒者、流浪修理工等临时商人 | 聚落日常生活 |
| **资源搜刮**：废墟探索、补给箱、紧急物资 | 长篇叙事、角色内心独白 |
| **聚落危机**：能源短缺、害虫入侵、聚落纠纷 | 已被"到处晃晃"覆盖的低风险日常 |

> **关键区分**：随机事件是唯一会在**行驶中主动打断**玩家的叙事通道。它的核心体验是"突发状况 → 紧急决策 → 承受后果"，不是"慢慢聊天"。如果一个事件没有风险、没有资源变动、只是增加关系值，它大概率不属于这里。

#### 驾驶闲聊 & 收音机 (chatter / radio)

| 负责 | 不负责 |
|------|--------|
| 旅途氛围、角色性格的碎片展示 | 任何需要玩家决策的内容 |
| 轻量世界观补充（收音机播报） | 推进主线、触发事件 |
| 不打断行驶的伴随式内容 | 大额奖惩、关系重大变化 |

> **设计原则**：「有就多赚一点，没有也不亏」——奖励关注但不惩罚忽视。

### 1.3 内容归属判断流程

编写新内容时，按以下顺序判断它应该放在哪个通道：

```
这段内容发生在什么场景？
│
├─ 开车途中，主动打断玩家 ─────────→ 有风险/资源变动？
│                                      ├─ YES → 随机事件
│                                      └─ NO  → 驾驶闲聊（改为不打断）
│
├─ 开车途中，不打断玩家 ──────────→ 驾驶闲聊 / 收音机
│
├─ 停车后，关于两人关系 ──────────→ 篝火对话
│
├─ 聚落内，和具名 NPC 交谈 ───────→ NPC 对话
│
├─ 聚落内，体验日常生活 ──────────→ 到处晃晃
│
└─ 推进世界线 / 引入新势力 ───────→ 主线对话
```

### 1.4 容易放错的内容类型

| 内容描述 | ❌ 容易放错到 | ✅ 应该放到 | 原因 |
|---------|-------------|-----------|------|
| "听流浪音乐家唱歌，+关系" | 随机事件 | 篝火对话 | 无风险无资源变动，纯关系内容 |
| "聊聊种植经验，+好感2" | 聚落随机事件 | 到处晃晃 | 低风险日常，属于聚落文化切片 |
| "参观工坊，+好感3" | 聚落随机事件 | 到处晃晃 | 聚落探索内容，不是危机事件 |
| "两人讨论刚遇到的铁锁商队" | 主线对话 | 篝火对话 | 是对已发生事件的"回响"，不推进新内容 |
| "韩策透露秘密情报" | 篝火对话 | 主线对话或 NPC 对话 | 有世界信息增量，有外部驱动 |
| "路上看到好看的云" | 随机事件 | 驾驶闲聊 | 纯氛围，不需要打断行驶让玩家做选择 |

### 主线对话的"三要素"检查

每条主线对话必须满足以下至少两项，否则应该归入篝火：

1. **外部驱动**：有来自主角之外的人物、势力或事件在推动场景发生
2. **世界信息**：玩家获得了关于世界状态的新认知（新势力、新线索、新危机）
3. **有重量的选择**：选项影响的不只是好感数值，还设置了影响后续的 flag 或改变某个 NPC/聚落的状态

**反面示例**（应归入篝火而非主线）：
```
❌ 两人在车里聊吃什么
❌ 陶夏吐槽林砾的习惯
❌ 看日落，感慨"信任是慢慢攒的"
❌ 翻账本互相调侃
```

**正面示例**（属于主线）：
```
✅ 到达温室时沈禾求助"帮我问塔台有没有水泵零件"（外部驱动 + 世界信息）
✅ 韩策冷淡地说"你们的运力不如铁锁稳定"（外部驱动 + 世界信息：铁锁存在）
✅ 路遇被拆除的路标，暗示静潮社活动（世界信息 + 悬念）
✅ 姜寒提出合并邀请，玩家必须选择（外部驱动 + 有重量的选择）
```

### 篝火对话中允许的"主线回响"

篝火对话可以**回应**主线事件（两人私下讨论刚发生的大事），但不应**推进**主线。区分：

```
✅ 允许（回响）：
"今天韩策说的那番话……你怎么看？"  ← 回应已发生的事
"路标被拆掉了，是谁干的？"         ← 讨论已发生的事

❌ 禁止（推进）：
"我刚收到消息，铁锁商队要来了"     ← 这是主线该做的事
"我决定去找陆沉谈谈"              ← 这是主线选择
```

## 二、每章世界状态速查表

### 核心定义

**已知 (KNOWN)**：玩家和主角都知道的信息，对话中可以自由引用。
**暗示 (HINTED)**：已出现了线索但未点明，可以模糊提及但不能明说。
**未知 (UNKNOWN)**：尚未有任何线索，对话中不得以任何形式出现。

### 序章（Chapter 0）

| 类别 | 状态 | 说明 |
|------|------|------|
| 温室社区 | 可能已知 | 取决于是否已到访 |
| 北穹塔台 | 可能已知 | 取决于是否已到访 |
| 废墟营地 | 可能已知 | 取决于是否已到访 |
| 钟楼书院 | 可能已知 | 取决于是否已到访 |
| 铁锁商队 | **未知** | 完全不存在于任何人口中 |
| 静潮社 | **未知** | 完全不存在 |
| 云海高塔 | **未知** | 不得提及 |
| 路标被挪 | **未知** | Ch.1 末尾才暗示 |
| 远方信号 | **未知** | Ch.2 才揭示 |
| 聚落间冲突 | **不存在** | 序章阶段各聚落互不干涉 |
| 两人的过去 | **未知** | 需要通过篝火慢慢揭示 |

**序章对话可以提及的**：
- 废土的日常生存（天气、路况、油量、食物）
- 行商的基础概念（买卖、运货、客户）
- 眼前看到的具体景象（残骸、废墟、远处的烟）
- 两人之间已建立的搭档关系（但不展开过去）

**序章对话不得提及的**：
- 任何尚未到访的聚落的具体信息
- 任何 NPC 的名字（除非玩家已经见过）
- "铁锁"、"静潮"、"商队联盟"等后期概念
- 云海高塔、远方信号、旧文明中枢

### 第一章（Chapter 1）

| 类别 | 状态 | 说明 |
|------|------|------|
| 4 个主聚落 | **已知** | 至少到访了 2 个，可以谈论 |
| 聚落领袖 | **已知** | 已到访的聚落领袖可以被提及 |
| 铁锁商队 | **暗示** | 仅韩策口中提到名字"铁锁商队"，但未见过 |
| 路标被挪 | **暗示** | Ch.1 末尾沈禾提到，但不知道是谁干的 |
| 静潮社 | **未知** | 不得出现 |
| 远方信号 | **未知** | 不得出现 |
| 聚落间关系 | **表面平静** | 各聚落有不同需求但无公开冲突 |

### 第二章（Chapter 2）

| 类别 | 状态 | 说明 |
|------|------|------|
| 4 个主聚落 | **已知** | 全部到访 |
| 铁锁商队 | **暗示→传闻** | 孟回提到"东线有大商队在签独家协议"，鸣砂广播提到"不明车队" |
| 远方信号 | **暗示** | 鸣砂广播提到"西南方向有塔还在发光"，林砾坦白追踪信号 |
| 静潮社 | **未知** | 路标被挪的事继续发生，但仍不知主体 |
| 聚落间关系 | **各有立场** | 四领袖对"远方信号"态度不同，但无冲突 |

### 第三章（Chapter 3）

| 类别 | 状态 | 说明 |
|------|------|------|
| 铁锁商队 | **已知** | 正面遭遇，姜寒和小铆登场 |
| 铁锁的行为 | **亲历** | 控价、收费关卡、合并邀请 |
| 静潮社 | **未知** | 仍然只有"路标被挪"的现象，不知主体 |
| 聚落间关系 | **开始紧张** | 铁锁的垄断行为影响了各聚落的贸易 |

### 第四章（Chapter 4）

| 类别 | 状态 | 说明 |
|------|------|------|
| 温室-塔台冲突 | **已知** | 水泵危机，公开对峙 |
| 铁锁 | **已知** | 可能作为调停方介入 |
| 霜期 | **暗示** | 首次短暂接触，但不知其身份 |
| 静潮社 | **暗示** | 霜期的存在暗示了某种组织，但未点明 |
| 林砾-陶夏分歧 | **发生中** | 是否介入聚落冲突的分歧 |

### 第五章（Chapter 5）

| 类别 | 状态 | 说明 |
|------|------|------|
| 静潮社 | **已知** | 正面接触陆沉，理解其动机 |
| 云海高塔 | **暗示→线索汇集** | 多方信息指向同一地点 |
| 四聚落关系 | **有裂痕但修复中** | 第四章冲突已解决 |

### 第六章（Chapter 6）& 终章

| 类别 | 状态 | 说明 |
|------|------|------|
| 云海高塔 | **已知** | 位置、用途全部揭示 |
| 所有势力 | **已知** | 铁锁、静潮均已和主角有过深入互动 |
| 四聚落协作 | **发生中** | 各聚落提供资源支持远征 |

## 三、跨聚落信息传递规则（防逻辑矛盾）

### 核心规则：聚落之间不会主动共享信息

废土世界的基本设定是**聚落之间缺乏信任、通讯不畅**。行商存在的意义就是"连接断裂的世界"。因此：

#### 规则 3.1：聚落领袖不知道其他聚落的内部事务

```
❌ 错误：
沈禾："韩策那边好像找到了一份旧地图，你去拿来给我看看。"
  → 沈禾怎么知道韩策找到了什么？她们之间没有通讯渠道。

✅ 正确（早期）：
赵苗："前几天有个路过的行商说，北边塔台那帮人在捣鼓什么老设备。"
  → 同势力的下属听到了传闻，来源是别的行商，合理。

✅ 正确（中后期，好感高了之后）：
沈禾："你上次帮韩策修的那个设备……他有没有提起过什么数据？"
  → 沈禾不知道韩策的事，但她知道玩家帮过韩策（因为玩家告诉她的）。
```

#### 规则 3.2：信息只能通过以下渠道传递

| 渠道 | 适用范围 | 限制 |
|------|---------|------|
| **玩家亲历** | 所有章节 | 玩家去过的地方、见过的人 |
| **同势力 NPC** | 同一派系的聚落之间 | 农耕派内部（沈禾↔赵苗）可以知道彼此的事 |
| **鸣砂广播** | 公共信息 | 只有鸣砂选择广播的内容才是公开的 |
| **行商传闻** | 模糊、不准确的二手信息 | "听说"、"有人提到"，不能是确切情报 |
| **孟回** | Ch.2 起 | 作为流动商人，他可以带来跨聚落的消息碎片 |
| **玩家主动告知** | 已建立信任后 | 需要好感达到一定等级，且需要显式的"我告诉了她"的叙事交代 |
| **雪冬信件** | NPC刻意写信传达的内容 | 信件内容受NPC信息权限约束（沈禾不会在信里写塔台技术数据）；详见 `letter_system.md` |

#### 规则 3.3：跨派系领袖直接通讯的阶段限制

| 阶段 | 领袖间的关系 | 对话中可以怎么写 |
|------|-------------|----------------|
| 序章~Ch.2 | **互不往来** | 领袖们知道其他聚落存在，但不了解对方近况 |
| Ch.3~Ch.4 | **通过玩家间接了解** | "你上次从塔台带回来的那批货……"（知道玩家跑商路线） |
| Ch.4 危机期 | **被迫对话** | 温室-塔台冲突期间可以有直接对话，但关系紧张 |
| Ch.5~Ch.6 | **开始协作** | 各聚落因共同目标开始有限度的沟通 |
| 终章 | **主动合作** | 四聚落各派代表，可以自由交流 |

#### 规则 3.4：具体 NPC 的信息权限

| NPC | 知道什么 | 不知道什么 |
|-----|---------|-----------|
| 沈禾 | 温室的农业、赵苗的近况、外围农场的事 | 塔台的技术细节、废墟的黑市、书院的档案 |
| 韩策 | 塔台的设备数据、程远的报告、信号频率 | 温室的种植计划、废墟的地下通道 |
| 伍拾七 | 废墟的地形、阿锈的消息、黑市的行情 | 塔台的技术、温室的储备量、书院的藏书 |
| 白述 | 书院的档案、苏墨的发现、旧文明的知识 | 各聚落的商业机密、铁锁的行动计划 |
| 孟回 | 各地传闻（模糊）、路况、市价 | 聚落内部事务、领袖的私下态度 |

## 四、角色说话方式规范

### 两位主角

| 角色 | 说话特点 | 禁忌 |
|------|---------|------|
| **林砾** | 话少、简短、偶尔冷幽默；用省略号多；情绪内敛，少用感叹号；技术话题时话会变多 | 不会长篇大论、不会主动表达感情、不会用夸张的语气词 |
| **陶夏** | 活泼、爱吐槽、爱用反问句；情绪外放但不聒噪；认真时语气会变沉 | 不是傻白甜、不会无脑乐观、面对真正的危险不会嘻嘻哈哈 |

**两人之间的互动规律**：
- 日常状态：陶夏主动发话 → 林砾简短回应 → 陶夏追问或吐槽 → 林砾偶尔一句让陶夏意外的话
- 严肃状态：角色反转，林砾主动开口时意味着她认真了，陶夏会安静下来听
- 分歧状态：不吵架，而是沉默——做饭时不说话、睡觉时背对着

### NPC 领袖

| NPC | 说话特点 | 关键词 |
|-----|---------|--------|
| **沈禾** | 务实、朴素、关心具体的事（水、种子、天气）；对外人冷淡但不恶意 | "够不够"、"能用就行"、"别浪费" |
| **韩策** | 理性、数据导向、对不专业的人没耐心；尊重能力不尊重人 | "数据"、"效率"、"不够稳定"、"你们的运力……" |
| **伍拾七** | 江湖气、直接、假装不在乎但其实讲义气；对规则和体制不信任 | "我的地盘"、"信你一次"、"规矩" |
| **白述** | 文雅、慢条斯理、爱引典故；温和但固执 | "档案里记载"、"旧时代"、"记住了就不会忘" |

### 重要准则

- **narrator（旁白）**只描述场景和动作，不表达主观判断
- **radio（鸣砂广播）**语气克制、信息密度高、偶尔带一丝人情味但从不煽情
- NPC 说话不应该像"RPG 任务发布器"——不要"请帮我做 XX，完成后奖励 XX"
- NPC 的请求应该融入自然对话，让玩家自己意识到"这是一个可以帮忙的事"

## 五、叙事调性红线

### 绝对不做

| 编号 | 红线 | 说明 |
|------|------|------|
| T-1 | **不写绝望结局** | 即使是最差的结局也是积极的，只是"积极的方式不同" |
| T-2 | **不写不可救药的反派** | 姜寒有道理，陆沉有苦衷。每个对手都是"用另一种方式回应恐惧的人" |
| T-3 | **不写角色死亡** | 这是温暖治愈底色的作品，主要角色和 NPC 不会死亡 |
| T-4 | **不写露骨的亲密场景** | 轻百合 = 暗示和余韵，不是直接描写。"陶夏靠近了一点点"比"陶夏牵住了她的手"更好 |
| T-5 | **不写主角被动接受命运** | 她们始终在主动选择，即使选择的是"现在不做选择" |
| T-6 | **不让战斗决定剧情** | 决定走向的是贸易、信任和选择 |
| T-7 | **不写超出世界观的科技** | 没有 AI、没有网络、没有卫星。通讯靠无线电、交通靠燃油车 |
| T-8 | **不写说教** | 主题通过角色行动展现，不通过角色演讲传达 |

### 调性标尺

```
←——绝望·残酷——|——紧张·严肃——|——平静·日常——|——温暖·治愈——|——甜蜜·煽情——→
                     ✅ 可以到这里        ✅ 主要在这里     ⚠️ 偶尔可以
                                                           ❌ 不要到这里→
```

- **大部分时间**：平静·日常 ~ 温暖·治愈
- **冲突时可以**：到达紧张·严肃，但不滑向绝望
- **关系高光时可以**：触碰温暖·治愈的上限，但不要变成偶像剧
- **绝对不要**：到达两端的极值

## 六、对话编写检查清单

### 写主线对话时

- [ ] 这条对话是否有外部驱动（NPC、事件、势力）？如果只有两人聊天，考虑移到篝火。
- [ ] 是否引入了新的世界信息？玩家看完后会不会对世界有新的认知？
- [ ] 选项是否有重量？至少一个选项应该设置 flag 或产生非好感的后果。
- [ ] 是否遵守了当前章节的信息权限？（对照第二节的世界状态表）
- [ ] 是否有 NPC 越权知道了不该知道的事？（对照第三节的信息传递规则）
- [ ] 林砾和陶夏的说话方式是否符合角色设定？

### 写篝火对话时

- [ ] 是否聚焦于两人的关系/情感/回忆？
- [ ] 是否避免了推进主线的内容？
- [ ] 如果引用了主线事件，是"回响"（讨论已发生的事）而非"推进"（触发新事件）？
- [ ] relation_stage 是否正确？early 阶段不应该有 late 阶段的亲密感。
- [ ] required_flags 是否合理？如果引用了某个主线事件，是否要求了对应 flag？

### 写 NPC 对话时

- [ ] NPC 是否只谈论自己知道的事？（对照 3.4 信息权限表）
- [ ] NPC 是否有跨聚落提供确切情报？如果有，信息来源是否合理？
- [ ] NPC 的好感门槛是否与信息价值匹配？重要情报应该需要高好感。
- [ ] NPC 说话方式是否符合角色设定？（对照第四节）

## 七、关系阶段的叙事约束

### 关系阶段定义

| 阶段 | 好感值范围 | 阶段标签 | 两人状态 |
|------|-----------|---------|---------|
| 初识 | 0~29 | early | 搭档、客气、偶尔拌嘴 |
| 默契 | 30~59 | mid | 习惯彼此、开始分享过去、有私下笑话 |
| 羁绊 | 60+ | late | 不需要说出口的信任、会为对方破例 |

### 各阶段的叙事允许/禁止

| 行为 | early | mid | late |
|------|-------|-----|------|
| 互相吐槽 | ✅ | ✅ | ✅ |
| 谈论日常分工 | ✅ | ✅ | ✅ |
| 分享过去的记忆 | ❌ | ✅ | ✅ |
| 讨论内心恐惧/脆弱 | ❌ | ⚠️ 小心处理 | ✅ |
| 肢体接触描写（拍肩、靠近） | ❌ | ⚠️ 极少 | ✅ 自然地 |
| "约定"类的承诺 | ❌ | ❌ | ✅ |
| 沉默中的默契（不说话也懂） | ❌ | ⚠️ 偶尔 | ✅ |
| 为对方改变自己的习惯/立场 | ❌ | ❌ | ✅ |

## 八、常见反模式（必须避免）

### 反模式 1：NPC 读心术

```
❌ 错误：
白述："你们最近在追踪一个信号，对吧？"
  → 林砾追信号的事只有陶夏知道，白述怎么知道的？

✅ 正确：
白述："你们行商跑的路线越来越远了。……在找什么？"
  → 观察到行为变化，自然的好奇，不是确切情报。
```

### 反模式 2：主线催促

```
❌ 错误：
narrator："你感到事态紧急，必须立刻出发前往塔台。"
  → 主线不催促玩家，节奏由玩家自己掌控。

✅ 正确：
沈禾："水泵的事……不急，但也不能一直拖着。你下次路过塔台的时候，帮我问一声吧。"
  → 有紧迫感但不强制，融入跑商节奏。
```

### 反模式 3：过早揭示

```
❌ 错误（Ch.1 对话中）：
陶夏："你说那些被挪的路标……会不会是某个组织干的？"
  → Ch.1 阶段不应该猜到静潮社的存在。

✅ 正确（Ch.1 对话中）：
陶夏："谁会闲着没事挪路标啊？又不是抢劫。"
林砾："……不知道。先记下来。"
  → 困惑但不推测，留下悬念。
```

### 反模式 4：旁白代入感情

```
❌ 错误：
narrator："这一刻，两人的心靠得更近了。"
  → 旁白不做主观情感判断。

✅ 正确：
narrator："沉默了一会儿。篝火噼啪作响。"
  → 只描述场景，让玩家自己感受。
```

### 反模式 5：选择无差异

```
❌ 错误：
选项A："好的。" → add_relation:2
选项B："嗯。"   → add_relation:2
  → 两个选项文字不同但效果相同，玩家选择无意义。

✅ 正确：
选项A："我们去帮忙。" → add_relation_taoxia:3, set_flag:chose_to_help
选项B："这是他们的事。" → add_relation_linli:2, set_flag:chose_neutral
  → 不同选择反映不同立场，影响后续。
```

### 反模式 6：角色降智

```
❌ 错误：
林砾："我完全没注意到铁锁在控价！"
  → 林砾是务实的机械师，她一定会注意到利润变化。

✅ 正确：
林砾："……利润比上个月少了两成。不是路线的问题。"
  → 角色能力范围内的正常反应。
```

### 反模式 7：好感断层

```
❌ 错误（early 阶段的篝火对话）：
陶夏把头靠在林砾肩膀上。
  → early 阶段两人还只是搭档关系，这种亲密度不合理。

✅ 正确（early 阶段）：
陶夏裹紧了斗篷，往篝火近了一点。
  → 自然的行为，不暗示超出当前关系的亲密。
```

## 九、技术格式规范

### 对话 JSON 结构

```jsonc
{
  "id": "SD_CH3_01",               // 前缀：SD_=主线，CF_=篝火
  "title": "铁锁的邀请",
  "type": "main_story",            // main_story / daily / memory / philosophy / cooperation
  "chapter": 3,                    // 所属章节
  "background": "image/xxx.png",   // 可选，对话背景图
  "node_types": ["settlement"],    // 触发位置类型
  "relation_stage": "any",         // any / early / mid / late
  "weight": 200,                   // 抽取权重
  "cooldown": 99,                  // 冷却（主线通常 99）
  "required_flags": ["ch2_done"],  // 前置 flag
  "forbidden_flags": ["sd_ch3_01_done"],  // 完成后禁止再次触发
  "steps": [
    // speaker: "narrator" | "linli" | "taoxia" | "radio" | NPC名
    { "speaker": "linli", "text": "……", "expression": "thinking" }
  ],
  "choices": [
    {
      "text": "选项文本",
      "ops": ["add_relation_linli:2", "set_flag:xxx"],
      "set_flags": ["sd_ch3_01_done"],  // 完成标记
      "result_text": "选择后的结果描述",
      "memory": { "id": "mem_xxx", "title": "标题", "desc": "描述" }
    }
  ]
}
```

### Flag 命名规范

```
主线完成：    sd_{chapter}_{seq}_done     # SD_CH3_01 → sd_ch3_01_done
章节完成：    ch{N}_done / prologue_done
篝火完成：    mem_{memory_id}             # 通常与 memory.id 一致
势力状态：    iron_lock_met / still_tide_softened
选择记录：    chose_{action}              # chose_to_help / chose_neutral
NPC 记忆：    mem_{npc}_{event}           # mem_shen_he_seeds
```

### expression 可用值

```
normal / happy / sad / surprised / angry / thinking
```

每个主角有对应的表情差分立绘。NPC 目前不支持 expression。

## 附录：快速参考卡片

### 我在写 Chapter N 的对话，应该注意什么？

1. 翻到第二节，找到 Chapter N 的世界状态表
2. 确认所有提及的信息都在"已知"或"暗示"范围内
3. 对照第三节，检查 NPC 的信息来源是否合理
4. 对照第四节，检查角色说话方式
5. 跑一遍第六节的检查清单

### 我不确定这条对话该放主线还是篝火？

问自己：
- 去掉两个主角的互动，这条对话还剩什么？
  - 如果剩下世界信息 / NPC 互动 / 事件推进 → **主线**
  - 如果什么都不剩 → **篝火**

### 我想让 NPC A 提到 NPC B 找到的东西，怎么写才合理？

1. A 和 B 是同势力？→ 可以直接提及（"程远那边说……"）
2. A 和 B 不同势力，当前章节 ≤ Ch.3？→ **不允许确切情报**，只能用传闻（"听路过的行商说……"）
3. A 和 B 不同势力，当前章节 ≥ Ch.4？→ 如果玩家曾告知 A 关于 B 的事，可以引用（"你上次提到的那个……"）
4. A 和 B 不同势力，当前章节 = Ch.6/终章？→ 协作阶段，可以直接交流
